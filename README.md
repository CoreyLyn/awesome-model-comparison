## â• Adding New Comparisons

### Add a New Prompt

1. Add a new prompt object to `prompts[]` in `data/prompts.json` (ensure unique `id`)
2. Create directory `html/<prompt-id>/`
3. Place each model's HTML output in the directory
4. Reference files in `models[].htmlPath`

### Add a New Model to Existing Prompt

1. Add a new model object to the target prompt's `models[]`
2. Place the corresponding HTML file
3. Update `htmlPath` to point to the file

## ğŸ”— URL Parameters

All state can be shared via URL query parameters:

| Parameter | Values | Description |
|-----------|--------|-------------|
| `prompt` | `<prompt-id>` | Select a specific prompt |
| `mode` | `grid` \| `split` \| `single` | Set view mode |
| `theme` | `dark` \| `light` | Force theme (omit for system default) |
| `q` | `<search-term>` | Pre-fill search query |
| `tags` | `<tag1>,<tag2>` | Pre-select tags (comma-separated) |

**Example:**
```
/?prompt=ios-weather&mode=split&theme=dark&tags=UI,iOS
```

### Preview shows "Loading..." indefinitely

1. Use a local static server (see "Getting Started" above) to avoid `file://` security restrictions

Ensure `data/prompts.json` is valid JSON and accessible. Invalid JSON will prevent the app from loading.
